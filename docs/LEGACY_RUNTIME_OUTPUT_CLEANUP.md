# Phase 68-70 Legacy Runtime Output Cleanup

## 1. Purpose

This document defines cleanup rules for legacy runtime outputs that were previously tracked by Git.

The project remains:

- research-only
- read-only
- manual-only
- no auto trade

## 2. Cleanup target

Legacy runtime files should not remain tracked by Git if they are generated by local manual runs.

They should be ignored by .gitignore and kept local only.

## 3. Commit policy

Remove from Git index:

- runtime CSV logs
- runtime Markdown reports
- Telegram-ready local text artifacts
- daily research run generated outputs

Keep committed:

- source code
- tests
- docs
- scripts
- public sample files

## 4. Safety boundary

This cleanup is Git hygiene only.

It must not trigger:

- broker execution
- real Telegram send
- real scheduler deployment
- automatic trading

## 5. Final boundary

Removing runtime outputs from Git tracking does not delete local files.

It only prevents generated artifacts from polluting future commits.
