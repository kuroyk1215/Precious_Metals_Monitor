# Telegram Pre-Integration Review

## 1. Purpose

This document defines the review requirements before connecting Telegram to Precious_Metals_Monitor.

Current boundary:

- research-only
- read-only
- manual-only
- no auto trade

Phase 35 does not implement real Telegram sending. It only defines the pre-integration design, safety boundary, configuration requirements, and failure handling model.

## 2. Integration goal

Telegram integration is intended to send research text notifications generated by the existing MVP workflow.

Allowed output:

- Telegram-ready research summary
- daily report summary
- final research trading plan summary
- data status notes
- manual review reminders

Not allowed:

- order placement
- order cancellation
- automatic buy/sell
- automatic rebalance
- broker instruction generation
- Telegram command that triggers trading behavior

## 3. Required Telegram components

Future real integration may require:

- Telegram account
- Bot created through BotFather
- Bot token
- chat_id or channel_id
- local private runtime config
- send permission check
- message formatting rule
- failure logging rule

## 4. Bot token handling

Bot token must never be committed to Git.

Allowed storage options:

- local config.yaml
- environment variable
- private runtime config outside Git tracking

Not allowed:

- examples/
- docs/
- README
- committed YAML files
- committed CSV files
- committed logs
- GitHub issues or PR descriptions

## 5. chat_id / channel_id handling

chat_id or channel_id should be treated as private runtime configuration.

It must not be placed in public sample configs.

A sample config may include placeholder values only.

Example placeholder:

    TELEGRAM_CHAT_ID_PLACEHOLDER

## 6. Message scope

Telegram messages should be limited to research and review content.

Permitted message categories:

- run completed
- report generated
- data unavailable
- delayed data warning
- final research plan ready
- manual review required
- scheduler run result

Forbidden message categories:

- buy now
- sell now
- place order
- cancel order
- rebalance now
- trade automatically
- any broker-side command

## 7. Manual review requirement

Every Telegram message must preserve the manual review boundary.

Required reminder:

    research-only / read-only / manual-only / no auto trade

The user must manually review the report before any trading decision.

## 8. Phase 35 non-goals

Phase 35 does not:

- connect to Telegram API
- send real Telegram messages
- store real Bot token
- store real chat_id
- add scheduler integration
- add auto trade logic
- change IBKR safety boundary
- change main.py
- change src/ trading logic

## 9. Future implementation gates

Before real Telegram sending is allowed, the project should define:

- token storage model
- chat_id validation
- message size limit
- failure logging
- retry policy
- dry-run mode
- send_allowed flag
- test mode
- no-trade assertion
