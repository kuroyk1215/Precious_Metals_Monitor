# Phase 65-67 Runtime Output Git Hygiene

## 1. Purpose

This document defines Git hygiene rules for daily research run runtime outputs.

The project remains:

- research-only
- read-only
- manual-only
- no auto trade

## 2. Runtime outputs

The following files are generated by local manual runs and should not be committed:

- daily_research_run_summary.csv
- final_research_plan_daily_log.csv
- final_research_plan_orchestrator.csv
- report_template_daily_log_telegram_ready_output.csv
- reports/daily_research_run_report.md
- reports/final_research_plan_orchestrator_report.md
- reports/report_template_daily_log_telegram_ready_output_report.md
- reports/telegram_ready_text.txt

## 3. Commit policy

Commit:

- source code
- docs
- scripts
- tests
- public sample files

Do not commit:

- runtime reports
- runtime CSV outputs
- local logs
- private config
- broker outputs
- tokens
- account data

## 4. Safety boundary

Runtime output hygiene is file management only.

It must not trigger:

- broker execution
- real Telegram send
- real scheduler deployment
- automatic trading

## 5. Final boundary

Daily research run artifacts are local review outputs.

They may be archived manually under ignored runtime paths, but they should not be committed by default.
